Modbus RTU协议与称重设备(PSD900)通信 代码理解

代码逐行解释

这段代码是一个C# Windows Forms应用程序，用于通过Modbus RTU协议与称重设备(PSD900)通信，并显示采集的数据。下面我将逐部分进行解释：

1. 命名空间和类定义

  • 引入了各种必要的命名空间，包括系统库、多线程、Windows窗体、Modbus通信库和自定义控件库
  • 使用了两个别名：ArrowDirection和ConfigHelper，使代码更简洁
  • 定义了一个名为Form1的窗体类，继承自UIForm(来自Sunny.UI库)
  • 声明了几个私有成员变量：
    • _modbusRtu: Modbus RTU通信对象
    • _ctsCollect: 用于取消异步任务的取消令牌源
    • _isCollect: 标记是否正在采集数据
    • _collection: 线程安全的阻塞集合，用于存储采集到的数据，最多保存500条

2. 构造函数和初始化方法

  • 窗体构造函数，在创建窗体时依次调用：
    1. InitializeComponent(): 初始化窗体组件(自动生成)
    2. InitConfig(): 初始化配置
    3. InitModbus(): 初始化Modbus通信
    4. InitUiRefresh(): 初始化UI刷新机制

2.1 UI刷新初始化

  • 启动一个后台任务不断检查_collection中是否有新数据
  • 如果有数据，通过Invoke方法在UI线程上更新界面控件：
    • ucMeter1和ucMeter2: 可能是仪表控件，分别显示数据的第0和第1个值
    • ucThermometer2: 温度计控件，显示数据的第3个值
    • ucArrowLoss: 箭头控件，根据第7个值显示方向(0=左，否则=右)
    • ucThermometer1: 另一个温度计控件，显示数据的第8个值
  • 每次循环后暂停10毫秒

2.2 配置初始化

  • 调用配置帮助类加载配置(如串口参数等)

2.3 Modbus初始化

  • 创建Modbus RTU通信对象
  • 配置串口参数(端口名、波特率、数据位、校验位、停止位等)
  • 设置其他Modbus参数(超时、地址格式、数据格式、站号、CRC校验等)

3. 按钮事件处理

3.1 连接按钮

  • 尝试打开Modbus连接
  • 成功时显示成功消息
  • 失败时显示错误信息

3.2 采集按钮

  • 检查是否已连接设备
  • 如果正在采集，则停止采集(改变按钮文本和颜色，取消任务)
  • 如果未采集，则开始采集(改变按钮文本和颜色)：
    • 创建新的取消令牌源
    • 启动后台任务每100毫秒读取一次Modbus数据(从站地址1，读取9个短整型)
    • 如果读取成功，尝试将数据添加到集合，如果集合满了则先移除一条再添加
    • 捕获并打印异常
  • 切换采集状态标志

3.3 模拟数据按钮

  • 模拟数据写入功能
  • 检查是否已连接设备
  • 创建新的取消令牌源
  • 启动后台任务每100毫秒生成随机数据并写入Modbus：
    • 生成随机系数、缺料状态和百分比
    • 创建包含9个短整型的数组
    • 将数据写入Modbus(从站地址1)
    • 捕获并打印异常

总结

这个程序的主要功能：

  1. 通过串口使用Modbus RTU协议与称重设备通信
  2. 提供连接设备的功能
  3. 提供数据采集功能，将采集的数据显示在UI控件上
  4. 提供模拟数据功能，用于测试

程序使用了多线程技术：

  • 使用BlockingCollection实现生产者-消费者模式，安全地在后台线程和UI线程之间传递数据
  • 使用CancellationTokenSource实现可取消的后台任务
  • 使用Task.Run和async/await处理异步操作

UI部分使用了第三方控件库(Sunny.UI和HZH_Controls)来美化界面，显示仪表、温度计和箭头等元素。
